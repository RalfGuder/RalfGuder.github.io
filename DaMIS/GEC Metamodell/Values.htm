C

context-element-association-description-code
Has-as-a-sub-CONTEXT-ELEMENT The value that links two context elements such that the subject CONTEXT-ELEMENT has as a sub-CONTEXT-ELEMENT the object CONTEXT-ELEMENT.
In-order-that The subject CONTEXT-ELEMENT is to be carried out so that the conditions are established for the completion of the object CONTEXT-ELEMENT.
In-response-to The value that links two CONTEXT-ELEMENTs such that the subject CONTEXT-ELEMENT is carried out in response to, or its performance is dependent on the completion of, the object CONTEXT-ELEMENT.
Is-a-modification-of The value denoting that the subject CONTEXT-ELEMENT amends (or suggests an amendment to) an existing object CONTEXT-ELEMENT.
Is-a-template-for The subject CONTEXT-ELEMENT constitutes an example that the object CONTEXT-ELEMENT should conform to.
Is-an-alternative-to The subject CONTEXT-ELEMENT may replace the object CONTEXT-ELEMENT.
Uses-as-a-reference The value that establishes a link between existing CONTEXT-ELEMENTs and new CONTEXT-ELEMENTs (Note: The subject CONTEXT-ELEMENT uses the specified object CONTEXT-ELEMENT as a reference).
control-element-classification-code
NATO-CONFIDENTIAL A specific value which denotes a certain REFERENCE-ELEMENT to be NATO CONFIDENTAL.
NATO-COSMIC-TOP-SECRET A specific value which denotes a certain REFERENCE-ELEMENT to be NATO COSMIC TOP SECRET.
NATO-RESTRICTED A specific value which denotes a certain REFERENCE-ELEMENT to be NATO RESTRICTED.
NATO-SECRET A specific value which denotes a certain REFERENCE-ELEMENT to be NATO SECRET.
NATO-TOP-SECRET A specific value which denotes a certain REFERENCE-ELEMENT to be NATO TOP SECRET.
NATO-UNCLASSIFIED A specific value which denotes a certain REFERENCE-ELEMENT to be NATO UNCLASSIFIED.
control-element-progress-description-code
ACCREDITATION-PERFORMED A data management progress information, which indicates that a certain data element is successfully accredited.
ACCREDITATION-REQUESTED A data management progress information, which indicates that a certain data element is requested for accreditation.
HARMONISATION-PERFORMED A data management progress information, which indicates that a certain data element is successfully harmonised in a specific data management project and sub process.
HARMONISATION-REQUESTED A data management progress information, which indicates that a certain data element is requested to be harmonised in a specific data management project and sub process.
MODELLING-PERFORMED A data management progress information, which indicates that a data element is sucessfully represented in a standard representation as a result of a certain sub process.
MODELLING-REQUESTED A data management progress information, which indicates that a certain data element is requested to be represented in a standard representation , e.g. as part of a relational data model.
POPULATION-PERFORMED A data management progress information, which indicates that a certain data element is successfully registered (populated) in the Information Repository, and ready for further processing.
POPULATION-REQUESTED A data management progress information, which indicates that a certain data element is requested to be registered (populated) in the Information Repository. This progress usually marks the beginning of the data management activities of a certain sub process of the data standardisation process.
STANDARDISATION-PERFORMED A data management progress information, which indicates that a certain data element is successfully standardised, and part of a new version of the Standard Data Elements.
STANDARDISATION-REQUESTED A data management progress information, which indicates that a certain data element is requested to be standardised.
VALIDATION-PERFORMED A data management progress information, which indicates that a certain data element is successfully validated.
VALIDATION-REQUESTED A data management progress information, which indicates that a certain data element is requested for validation
VERIFICATION-PERFORMED A data management progress information, which indicates that a certain data element is successfully verified.
VERIFICATION-REQUESTED A data management progress information, which indicates that a certain data element is requested for verification

D

data-reference-element-description-code
ASSOCIATIVE-GENERIC A generic word of a STANDARD SCHEME which denotes an interrelating category.
ATTRIBUTE A certain aspect or property of an ENTITY.
CHARACTERISTIC-DATA Empirical, synthesized, or otherwise provided parameters describing the characteristics of the system or component being modeled or simulated, (e.g., gross vehicle mass, gear ratio, terrain resistance coefficient).
COMPLETE-SUBTYPING-RELATION A reference of a complete subtyping relation of a specific data model.
DATA-MODEL A reference of a data model as a whole.
DEPENDENT-ENTITY A reference of a dependent entity of a specific data model.
DOMAIN A reference of a logical data type.
ENUMERATED-GENERIC A generic word of a STANDARD SCHEME, which denotes a characteristic feature represented by predefined values.
ENUMERATION A reference of permitted values (ENUMs).
FILE A reference of a data file as a whole.
FILE-FIELD An elementary component of a FILE-RECORD and caracterised by the assignment of a certain value.
FILE-RECORD A component of a FILE, which combines other records or fields to adequately describe a dataset.
IDENTIFYING-RELATION A reference of an identifying relation of a specific data model.
INCOMPLETE-SUBTYPING-RELATION A reference of an incomplete subtyping relation of a specific data model.
INDEPENDENT-ENTITY A reference of an independent entity of a specific data model.
NON-IDENTIFYING-RELATION A reference of a non-identifying relation of a specific data model.
NOTIONAL-DATA Speculative or theoretical data rather than actual data.
ORIGINAL-DATA The source data used by a resource producer to construct their initial abstract representation.
PRIME An an individual information domain as part of a standard scheme.
STANDARD-SCHEME A scheme of Standardised Data Elements as a whole.
TECHNICAL-DATA Scientific or technical information recorded in any form or medium (such as manuals and drawings).
VALIDATION-DATA The real-world facts used for comparison to validate the results of a simulation. These data come from empirical sources such as test ranges, live exercise results, or historical records, from outputs of other simulations that were previously validated, or from the knowledge of subject matter experts.
VALUE-GENERIC A generic word of a STANDARD SCHEME, which denotes a characteristic feature, but not represented by predefined values.

M

metadata-element-establishment-usage-code
CONDITIONAL A specific value which denotes a certain data establishment element to be of conditional use. As an example, a data establishment of an ENTITY and an ATTRIBUTE, which is characterised to be 'conditional', indicates that an attribute value may be set under certain conditions.
MANDATORY A specific value which denotes a certain data establishment element to be of mandatory use. As an example, a data establishment of an ENTITY and an ATTRIBUTE, which is characterised to be 'mandatory', indicates that an attribute value must be set.
NOT-APPLICABLE A specific value which describes, that it is not applicable to assigned a certain usage indicator to a certain data establishment element.
NOT-DEFINED A specific value which describes, that there is no usage indicator assigned to a certain data establishment element.
OPTIONAL A specific value which denotes a certain data establishment element to be of optional use. As an example, a data establishment of an ENTITY and an ATTRIBUTE, which is characterised to be 'optional', indicates that an attribute value may be set.

P

post-element-role-code
ACCREDITATION-AGENT The organisation designated by the M&S USER to conduct an accreditation assessment for an M&S application.
CONFIGURATION-AGENT A representative of a data management organisation, who is responsible for configuration management.
CONSULTANT A data management and modeling expert, who supports representatives of the Data Management Organisation in certain technical aspects.
HARMONISATION-DATA-AGENT A representative of a data management organisation, who is responsible for a data harmonisation activity within the data standardisation process.
HEAD-OF-INSTANCE A representative of a data management organisation, who is responsible head of a certain data management instance. A data management instance encloses all the different data management units, and is established for a certain service or functional area of the German Armed Forces.
HEAD-OF-UNIT A representative of a data management organisation, who is responsible head of a certain data management unit.
MODELLING-DATA-AGENT A representative of a data management organisation, who is responsible for a data modeling activity, e.g. for the representation of standardisation results as integral parts of the Core Data Model.
POPULATION-DATA-AGENT A representative of a data management organisation, who is responsible for a data registration or population activity.
PROCESS-AGENT A representative of a data management organisation, who is responsible for a certain sub process of the data standardisation process.
PROJECT-AGENT A representative of a data management organisation, who is responsible for a certain project within a sub process of the data standardisation process.
QUALITY-AGENT A representative of a data management organisation, who is responsible for quality management.
STANDARDISATION-DATA-AGENT A representative of a data management organisation, who is responsible for a data standardisation activity.
SYSTEM-AGENT A representative of a data management organisation, who is responsible for or point of contact of a certain IT system.
VALIDATION-AGENT An organisation designated to perform validation of a model, simulation, or federation of models and/or simulations and the associated data.
VERIFICATION-AGENT An organisation designated to perform verification of a model, simulation, or federation of models and/or simulations and the associated data.

S

system-reference-element-description-code
COMMAND-AND-CONTROL-SYSTEM [No definition inserted].
CONSULTATION-SYSTEM [No definition inserted].
SIMULATION-SYSTEM A SIMULATION-SYSTEM represents the sum of all software resources, algorithms and procedures, which are required to perform a simulation or a part of simulation.
system-reference-element-interface-indicator-code
INPUT A specific value which indicates an certain data flow as an input data flow. For example a certain data element may be used as an input data field in a specific system component or method.
INPUT-AND-OUTPUT A specific value which indicates an certain data flow as an input and output data flow. For example a certain data element may be used as an input and output data field in a specific system component or method.
NOT-DEFINED A specific value which indicates that no data flow information is assigned.
OUTPUT A specific value which indicates an certain data flow as an output data flow. For example a certain data element may be used as an output data field in a specific system component or method.

topic-map-association-description-code

Is-a-modification-of The value denoting that the subject TOPIC-MAP amends (or suggests an amendment to) an existing object TOPIC-MAP.
Is-a-template-for The subject TOPIC-MAP constitutes an example that the object TOPIC-MAP should conform to.
Is-an-alternative-to The subject TOPIC-MAP may replace the object TOPIC-MAP.
Uses-as-a-reference The value that establishes a link between existing TOPIC-MAPs and new TOPIC-MAPs (Note: The subject TOPIC-MAP uses the specified object TOPIC-MAP as a reference).

U

unit-element-role-code
ACCREDITATION-AUTHORITY An organisation responsible to approve the use of a model, simulation, or federation of simulations for a particular application.
ACCREDITATION-SPONSOR An organisation with the responsibility for accrediting a model, simulation, or federation of models and/or simulations for a specific use or series of uses.
CONFIGURATION-MANAGEMENT-UNIT A unit and part of a certain data management instance, whose representatives are trained for, and perform configuration management activities in the data standardisation process.
DATA-CONSUMER An organisation, which has a certain need for data elements, which is under control of a data management organisation.
DATA-HARMONISATION-UNIT A unit and part of a certain data management instance, whose representatives are trained for, and perform data harmonisation activities in the data standardisation process.
DATA-MANAGEMENT-INSTANCE A certain data management instance.
DATA-MODELLING-UNIT A unit and part of a certain data management instance, whose representatives are trained for, and perform data modeling activities in the data standardisation process.
DATA-POPULATION-UNIT A unit and part of a certain data management instance, whose representatives are trained for, and perform data population activities in the data standardisation process.
DATA-PRODUCER The organisation responsible for gathering or developing data and assessing their quality.
DATA-PROPONENT The organisation that has primary responsibility for a data collection or database. The data proponent develops the requirement for the data.
DATA-SPONSOR An organisation, which provides a certain data management organisation with data elements.
DATA-STANDARDISATION-UNIT A unit and part of a certain data management instance, whose representatives are trained for, and perform data standardisation activities in the data standardisation process.
INFORMATION-MANAGEMENT-UNIT A unit and part of a certain data management instance, whose representatives are trained for, and perform information management activities. As an example, the information management unit takes care of an adequate use a the standardised data elements.
INTERFACE-MANAGEMENT-UNIT A unit and part of a certain data management instance, whose representatives are trained for to keep contact to other national or international data management and administration organisations.
M&S-DEVELOPPER An organisation that actually develops a model or simulation or the agency that is overseeing the M&S development by a contractor or federally funded research and development center.
M&S-USER An organisation that employs or will employ a model, simulation, or federation, its products, or its services to achieve a set of objectives. The User may also be involved in the evolution of such products and services.
ONTOLOGY-MANAGEMENT-UNIT A unit and part of a certain data management instance, whose representatives are trained for, and perfom data management activities which concentrate on the establishment and administration of a common ontology built out of the standard data elements.
QUALITY-MANAGEMENT-UNIT A unit and part of a certain data management instance, whose representatives are trained for, and perform quality management activities in the data standardisation process.
